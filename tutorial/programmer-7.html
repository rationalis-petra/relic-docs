Relic has three 'levels' of typechecking, which can be set at the level of the module or package. They are:

Dynamic
All typechecking is done dynamically. If a symbol is annotated with a type, then trying to assign it a value with a different type produces a warning (but not an error). Trying to use it at runtime as if it had a different type is an error.
Optional
Typechecking is only done at compile time on symbols which have been annotated.
Static
Typechecking is always done at compile time. If no annotation is present, the system will attempt to infer the type.
(ann my-name String)
(def my-name "Alice")

(ann add Fn (Int Int) Int)
(def add fn [x y] + x y)


;; Will fail! - proc implicitly sequences actions
(def add proc [x y] + x y)
;; ok :)
(def add proc [x y] (+ x y))

Relic has subtyping, which follows the rules you would expect: 

(ann get-x Fn (Struct (x Int)) x)
(def get-x fn [s] s.x)

;; subtyping!
(get-x (struct (x 3) (y "hi")))

Relic has System-F like types 

(ann id All [A] Fn [A] A)
(def id fn [x] x)

Relic can also infer, given sufficient annotation within a term, e.g. 

(def id-2 all [A] fn ([x A]) x) ;; infers as All [A] Fn [A] A

Finally, there is typing of delimited continuations, which have the form 

(ann eff Delimit Bool Int)
(def eff (raise true 2))

(handle (list (control (:amb 1 2)))
  (fn rec [v k] match v
    (:fail -> (list))
    (:choose v1 v2 -> append (handle (k v1) rec) (handle (k v2) rec))
