Current Directory

Here, we will focus on adding some simple features to our shell. First up, it is common for a shell to print the name of the current directory before the prompt. In Relic, we can access this string with get-current-directory, which will return a (freshly allocated) string with the name of the current directory. As with read-line, we will need to free this memory later. Let's replace our call to write-string with a prompt function which will print the prompt for the user. 

(def prompt proc [] seq
  [let cwd (platform.filesystem.get-current-directory)]
  (write-line cwd)
  (data.string.delete cwd)
  (write-string "> "))

Inside our main function, we then can rewrite the loop. 

  (loop 
    (prompt)
    [while (iter (read-line))])

Before moving on, now is a good time to start thinking about imports referring to the get-current-directory function by the fill import path is a bit unwieldy, so we can import the filesystem module, meaning that we can simply refer to filesystem.get-current-directory. 

(import platform.filesystem)

;;....
 
(def prompt proc [] seq
  [let cwd (filesystem.get-current-directory)]
  (write-line cwd)
  (data.string.delete cwd)
  (write-string "> "))

Commands

Of cource, once we display the current directory, we will want a way to change it. This can be done in relic using set-current-directory, but before we can do that, we first need to provide our shell with the ability to recognize different commands, beyond just 'exit'.

The natural next step then is to separate our input string on spaces, so let's start by importing some functions that will be useful later. 

(import
 platform.filesystem
 platform.memory

 (data.pointer :only (get set local))
 (data.list :only (mk-list push elt))
 (data.string :only (subview nth-byte)))

Our algorithm will run as follows:

  • Initialize the compoents (list of strings we want to return) to an empty list
  • Initialize the start of the current word to the start of the string

Then, we will loop over each character in the string, and:

  • If current character is a space, do to things:
    • Add a new word starting at our most recent word start, and ending just before the current character.
    • Reset the start of the current word to the character after the current one

Note that the last character may NOT be a space, so once our loop finishes, we will need to act as if there is an extra character that is a space at the end of the loop, to ensure the last word gets pushed on.

This algorithm is not perfect, and handles, e.g. double spaces less than ideally, however, it is nice and simple, and should be more than enough for our purposes.

We will start with our initialization. In Relic, the only way to get mutable values by mutating memory explicitly, i.e. via pointers. We can use the local function that we imported from the pointer module to create a pointer using local (i.e. on-stack) memory. Note that you should never return a local pointer from a procedure, as local (stack) memory is invalid once the procedure returns.

We can initialize start with 0, and we want to initialize our list with 0 elements, but with space reserved for 8. 8 has been chosen as it will be enough for most cases without being excessive, as lists will automatically grow if we exceed their capacity. 

(def split-on proc [str delimiter] seq
  [let! start (local 0)]
  [let! components (local (list.mk-list 0 8))]

  ;; Rest of the function goes here...
  )

The body of the loop is relatively easy to fill out, and we will make use of a new piece of syntax and some new functions. The when conditional is like if, but has no :false branch, as it is assumed implicitly return the value :unit. Additionally, like proc and def, it performs auto parenthesization after the condition.

Our new functions are push which will add a value to the end of our list, and subview, which is like substring, but does not allocate any memory, it simply provides a substring which views the original memory. This will make cleanup easier later. The arguments to subview are the start and end position of the substing, together with the original string we want to take the substring of. 

(def split-on proc [str delimiter] seq
  [let! start (local 0)]
  [let! components (local (list.mk-list 0 8))]

  (loop [for i from 0 below str.memsize]
    (when (= (nth-byte i str) delimiter) seq

      ;; Push the subcommant 
      (push (subview (get start) i str) components)
      ;; Set the start of the next womd to the character after the current one
      (set start (+ i 1))))
  ;; Rest of the function goes here...
  )

To finish the function, we will want to do the final subview push, to treat the character after the last character as a space, then we will want to return the value in the components pointer, which we can do with the get function. 

(def split-on proc [str delimiter] seq
  [let! start (local 0)]
  [let! components (local (list.mk-list 0 8))]

  (loop [for i from 0 below str.memsize]
    (when (= (nth-byte i str) delimiter) seq

      ;; Push the subcommant 
      (push (subview (get start) i str) components)
      ;; Set the start of the next word to the character after the current one
      (set start (+ i 1))))

  (push (subview (get start) str.memsize str) components)
  (get components))

Now that we have our split-on function, we can re-write our iter function, by getting the commands from the input line, then printing out each, which will be a useful diagnostic to see if our procedure works correctly. To write a character literal in Relic, we use #<char>, so #a is the character literal for a. Note that in Relic, there is no difference between a character and a function, so #a is the same as the number literal 97. For some 'special' characters, we refer to them by name, so space is #space.

Once we are done with them, we can free the memory associated with the underlying list with memory.free. 

(def iter proc [str] seq
  [let! commands (split-on str #space)]

  (loop [for i from 0 below commands.len]
    (write-line (elt i commands)))

  [let! should-exit (if (= str "exit") :false :true)]
  (memory.free commands.data)
  (string.delete str)
  should-exit)

Advanced Control Flow

Now that we have the command components, we will want to implement some commands. We wil start with cd to set the current working directory, as promised earlier, and echo, as it is simple. We will start with echo and function to print errors if we receive a command that isn't recognized: 

(def unrecognized proc [cmd] seq
  (write-string "Unrecognized command: ")
  (write-line cmd))

(def echo proc [args] seq
  (loop [for i from 1 below args.len]
    (write-string (elt i args))
    (write-string " "))
  (write-string "\n"))

Next, we will cover the cd command, and use a new error handling strategy making use of the labels term former. labels allows for code somewhat similar to using goto in C, but with several differences.

The general layout of a labels form is as follows: 

(labels
  <starting-expression>
  [label-1 [params...] <body>]
  [label-2 [params...] <body>]
  ...)

The way a labels statement works is that, whenever you are evaluating in either the starting-expression or one of the label bodies, you can invoke (go-to label args...) to jump execution to that label, with the label's params substituted for the go-to's args. As a simple example, we can set up a very basic loop that will print the numbers 1 to 10, shown below.

We have only two labels, loop-continue, which performs one iteration of the loop, and loop-exit, which is how we stop the loop. The starting expression is very simple and just kicks off the loop. Feel free to copy-paste this example into a repl, or a .rl file and play around with running different variations. 

(labels
  (go-to loop-continue 0)

  [loop-continue [i] seq 
    (write-string "i: ")
    (write-line (to-string i))

    (if (u64.< i 10)
        (go-to loop-continue (+ i 1))
        (go-to loop-exit))]

  [loop-exit (write-line "done")])

For our cd procedure, we will use labels to help in our error handling. We have only one label: the early-exit label, which lets us quit the proc and print an error message if something goes wrong. Otherwise, our starting expression is actually most of the function, and represents the 'happy' (error free) path for our code. At the moment, we will leave handling any errors raisd by set-current-directory for later. 

(def cd proc [args] labels
  (seq
    (when (u64.< args.len 1)
      (go-to early-exit "error: cd requires an argument; none was provided"))
    (when (u64.> args.len 2)
      (go-to early-exit "error: cd requires exactly one argument; too many were provided"))

    (write-string "changing dir to: ")
    (write-line (elt 1 args))
    (filesystem.set-current-directory (elt 1 args)))

  [early-exit [reason]
   (write-line reason)])

Now that we have our 3 commands, we can re-write the iter proc once again. At the moment, the simplest implementation is just to compare the command againts all known command strings, i.e. "exit", "cd" and "echo". Normally, you would do this with an if/else-if chain. However, the structure of Relic if would make this unnecessarily painful to write, i.e. (if <cond> <case1> (if <cond2> <case2> ...)), we would stack up a number of closing parentheses equal to the number of if cases. Yikes! Instead, Relic has cond, which lets you write a set of [condition body] pairs, and it will match on the first condition that returns true. A quirk all terms in Relic needing to evaluate to a value is that cond must guarantee that at least one of the branches matches. As such, the last condition in a cond must always be the literal :true, and you will get an error otherwise. 

(def iter proc [str] seq
  [let! should-continue local :true]
  [let! commands (split-on str #space)]
  [let! command-name (elt 0 commands)]

  (cond
   [(= "exit" command-name) (set should-continue :false)]
   [(= "cd" command-name) (cd commands)]
   [(= "echo" command-name) (echo commands)]
   [:true (unrecognized command-name)])

  (string.delete str)
  (memory.free commands.data)
  (get should-continue))
